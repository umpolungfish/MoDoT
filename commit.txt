Add the alchemical volatility bench: distill, fdistill, sublime

The modern bench (click, set, scan, polymerize, forge, arrange, anneal,
cleave, fuse, dope) went in without the classical alchemists' separation
tools. This starts filling that gap. The semantics were fixed first by an
ob3ect batch (alchemical_tools_roadmap): every classical technique typed
as the SAME Frobenius shape, a feedstock FSPLITs into a kept arm (EVALT)
and a rejected arm (EVALF) and the arms FFUSE back to the feedstock (mass
balance), verdict PASS, tier O1. So the whole bench is one fractionate
core parameterized by a per-unit axis, not eleven unrelated verbs.

This commit lands the volatility family, all three on Criticality (the
index-8 axis the oracle put the EVALT gate on, the readiness to leave):
  - distill M1 M2...   cut the set at the midpoint of the volatility range
    into the volatile head (distillate) and the involatile residue
    (bottoms); a pair tied on the axis within epsilon is an azeotrope
    (the B frontier) the cut cannot resolve.
  - fdistill M1 M2...  the fractional, higher-resolution version: rank the
    whole column plate by plate with the resolution gap to each next
    fraction, flagging neighbors too close to resolve (they co-distill;
    more plates will not split an azeotrope).
  - sublime A          purify one unit by a two-state skip across the axis,
    omitting the middle register; reports whether it sublimes or is
    entrapped within a step of the ceiling and must climb stepwise (excite).

Shared fractionate() core in click.rs, wired through the Cli flags, the
agent verb whitelist (run_structural_tool + verb_usage + STRUCTURAL_VERBS
+ TOOLS_PROMPT + the no-op-prod cues), with unit tests for the cut and the
azeotrope frontier. Verified live on real catalog entries.
