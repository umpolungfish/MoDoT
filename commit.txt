I built the excription learning loop and the batch admission gate.

imasm learn <word> [rounds=N] [breadth=K] (ask_native/src/learn.rs): one model
takes a word, iterates its grammar-valid single-edit neighborhood (boundary
turnstiles held fixed), excribes each candidate into a standalone domain object
that never names an opcode, imscribes the OBJECT back into a word, checks both
words, and measures the residual as token edit distance. Residual zero is the
round trip closing over the model itself. Aligned confusions persist atomically
to ob3ects/imasm_knowledge.json, distill into lessons that ride the imscriber's
next prompt, and the walk moves to the highest-residual candidate: the loop
spends its rounds at the frontier of its own ignorance. Guide gains Part XIV.

The batch admission gate (main.rs): a round's tool calls are judged AS A BATCH
before any runs. One improper call (unknown verb, pasted ellipsis, or a name
absent from the catalog) refuses the whole round unrun, each call's verdict is
spoken back, and the agent resubmits properly formatted. A missing imscription
is initiated BY THE GATE (the same procedural mint the imscribe verb runs,
capped per round against mint runaways), so the reissued call finds its entry.
Separators, key=value options, numbers, glyphs, and the twelve axis words are
never misread as name slots; near-matches and decoration/ladder bases pass.
