Bound the transient-retry so it can't churn, and add a --think toggle

Two things.

1. Stop the "stuck churning". My previous retry-on-transient change could
   turn one slow call into several: a read that hung its way to the 300s
   socket deadline returns "timed out reading response", which counts as
   transient, so it retried up to twice more — ~15 minutes of hanging on a
   single prod step. The retry now only fires when the failure came back
   FAST (under ~45s * attempt): a quick connection reset still retries, but
   a call that hung to a timeout is let stand instead of being tripled.

2. Add the ability to toggle model reasoning. New `--think` (on by
   default, `--think false` / `--no-think` to disable, MODOT_THINK env to
   set the default). When off it sends the provider's real disable-
   reasoning parameter: OpenRouter `reasoning.enabled=false`, Gemini
   `thinkingConfig.thinkingBudget=0`. Threaded through Llm.think and both
   inference bodies; no-op on models that don't support reasoning.

Full suite green (26 passed); all flag forms parse.
