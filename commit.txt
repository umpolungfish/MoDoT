MoDoT: print the spectrum in the order the gap is computed in

The spectrum was printed sorted by SIGNED value while the gap used MAGNITUDE
order, so the two disagreed about which eigenvalue is lambda_2. A reader taking
lambda_2 positionally -- the standard convention, lambda_1 >= lambda_2 >= ... --
read the wrong one.

Live: spectrum [+3.372, -1.000, -2.372], gap reported 1.0000. The model applied
the correct formula, rho - |lambda_2|, to the number our output told it was
lambda_2, got 3.372 - 1.000 = 2.372, and then wrote "-> 1.0000" -- silently
correcting to our answer without noticing its lambda_2 was ours in a different
order. It was right about the formula and wrong about the value because the
output taught it the wrong value.

The spectrum now prints by descending |lambda|, so positional lambda_2 IS the
one the gap uses, and the label says so. Verified: [+2.618, -2.618, +0.382,
-0.382] with gap 2.618-2.618 = 0.0000 -- the naive read and the computed value
now agree.
