Make --cycles actually compound: each cycle builds on the last

Cycles already threaded the prior cycle's final answer into the next cycle's
context, but nothing told the agent to BUILD on it -- it re-asked the same seed
with the prior answer sitting passively in history, so it was as likely to
re-derive as to continue. The compounding was in the data, not the instruction.

Now every cycle after the first opens with an explicit directive: begin from
where the last cycle ended, take its conclusions as the new starting point, push
further (pursue what it left open, deepen or stress-test what it found, build the
next layer), and do NOT re-derive what the last cycle already settled. Each cycle
is a deeper breath, not a repeat. The original seed/question is kept below for
reference, marked as reference rather than a fresh prompt. Works for both a normal
answer and a jam (a jammed cycle 2 continues the threads cycle 1 opened).
