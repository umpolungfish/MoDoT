Read strain and spectral gap jointly: the clarity/stability line

Lando's observation, now encoded: a shaky-but-workable proof forges a ring
with strain AND a non-zero spectral gap; as the object stabilizes the
strain drains out, the gap collapses to 0, and the statements it supports
go clarion. The material sheet was misreading that endpoint. It stamped a
zero gap as FRUSTRATED and the agent took it as a B-frontier negative,
which is the transport frame being pejorative about the exact state a proof
reaches when it clarifies (a flat, degenerate top spectrum = no privileged
mode = maximal symmetry, the SIC tight-frame geometry).

Fixes, all in the material sheet:
  - the spectral-gap line now reads a zero gap as a flat/degenerate top
    spectrum, NO privileged mode (the symmetric, settled reading), not a
    deficiency;
  - the "frustrated" conductance is relabeled BALANCED and marked not a
    defect on its own (defer to the clarity line + strain);
  - a new clarity line reads the (strain, gap) pair JOINTLY:
      both ~ 0        -> SETTLED / CLARION (the stabilized endpoint, the goal)
      relaxed, gap>0  -> STABLE BUT LEANING (leans on one mode)
      flat, strain>0  -> SYMMETRIC BUT STRESSED (anneal to settle)
      both > 0        -> SHAKY (where proofs come out shaky)
    Only the joint zero is clarion, per the principle.

ring_spectral_gap() shares the gap number between the spectrum print and
the clarity reading. Verified live: the Erdos AP ring (sigma=0.167, gap=0)
reads SYMMETRIC BUT STRESSED, and a tight pair (sigma=0, gap=0) reads
SETTLED / CLARION.

Aside (why this pair and not the SIC residual): the canonical SIC closure
residual frobenius_closure_residual was tested and is identically 0 for
every state, because a valid SIC reconstructs any density matrix exactly
(informational completeness). It cannot discriminate per object. The
(strain, gap) pair does, so that is the clarity certificate.
