Every catalog entry has its IMASM word, and the loop can start from a name.

An entry's twelve types are each a full program, so its word follows from
the catalog with no model asked. `imasm words` materialises the whole book
in one pass, and `imasm learn <entry>` seeds from a name by reading it.

The point is not convenience. A catalog entry comes with the object it IS,
so an excription from that seed has a truth to be judged against and not
only a residual to be measured. The loop has been scoring itself against
its own recovery; from a named seed it can be scored against the catalog.

One thing had to be fixed to make a named seed work at all. A composite
word carries twelve sources and twelve sinks, so judged as ONE word it is
ill-typed, every single-glyph neighbour of it is ill-typed too, and the walk
halted on an empty neighbourhood. Validity is now segment-aware: a word of
several programs is valid exactly when each of its programs is. Single
programs are judged as before.

Help text carries all of it: the wordbook verb, the entry seed, and what
seeding from an entry buys.
