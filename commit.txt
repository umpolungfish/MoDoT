Nature chemistry, and the layer where type identity actually lives

Add value-layer co-typing and the paradice as its unit: compare two
natures axis by axis over the twelve families, agreement settles an axis,
disagreement holds both at once as one paradice. cotype() and paradices()
invent nothing, they only read values the ob3ects already fixed. I did
not touch the vessel's Belnap-code layer, because mapping a family value
to a B4 code needs a principled rule and injecting a guess would break
the vessel's no-fallback contract.

Building it surfaced the real structure of the corpus, so I followed it.
The endpoint tuple I first loaded is near-degenerate: the 49 types fall
into two value-tuple classes, and the flat phase_1 opcode word is one
class for all of them. What separates the types is the derivation path,
not its endpoint. So Nature now carries a Signature read off the scaffold
header (sig, period, self_ref, frobenius_order, fork/fuse pairs, tier),
which is the composer's TokenFingerprint. distinction_layers() states the
resolution of each descriptive layer plainly: value tuple 2, opcode word
1, path signature 40, surface tokens 49. Type identity is fully carried
at the semantic surface and nearly at the path signature; the value and
opcode layers are ground states shared across the set.

This relocates the gate order. The composer/tier layer is not the last
plank to add but the one where type distinction already lives, so co-
typing that discriminates types must read the path signature, not the
value tuple. Registry gains by_tier() and signature_classes(). Exposed
cotype, paradices, CotypeReport, Signature at package level. Self-test
asserts the path signature out-resolves the value tuple.
