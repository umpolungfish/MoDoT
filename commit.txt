Loosen answer_is_proof so the lane guard fires on real proofs

A jam produced a proof-shaped answer that wrote "### The Proposition" (no
colon) and "Conventional Proof", but answer_is_proof required "proposition:"
WITH a colon AND a separate proof word, so it returned false. The lane
guard therefore did not abstain the material non-closure (tools=F) on that
proof, and the spine held fused=B (conflict=2) where it should have read
the tools as N (a forge that terminated does not deny an analytic proof).

Loosen the detector to any single strong proof-structure marker:
proposition / theorem / lemma / conventional proof / proof: / q.e.d / ∎.
This is the conservative direction (it only ever abstains a material F on a
proof, never the reverse), so a casual "proof" mention at worst declines to
let a forge veto an answer. Regression test added for the no-colon
"Proposition" / "Conventional Proof" shape that slipped through.
