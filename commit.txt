Ungag the agent: the mint was being eaten, and prove now takes an entry

Three faults, one story. A jam cycle on the perfect cuboid kept emitting
`imscribe perfect_cuboid_descent_reagent "…"`, the next `click` kept answering
"catalog entry not found", and it looped. I read the first tool_calls log
yesterday and blamed the agent: treating a frontier as a wall, the GENERATIVE
POWER clause not taking. Wrong. It reached for the mint every time and the tool
layer swallowed the call.

The prose guard was the gag. It refuses any arg containing `(`, `)`, a backtick
or `:` — correct for `TOOL: ascend again`, fatal for the only two verbs whose
tail argument IS free text. `imscribe` joins args[1..] into a description and
`ob3ect` takes a whole natural-language spec, so the better the agent specified
the object it wanted to create, the likelier the colon and the parens, and the
more certainly its creation was discarded. Silently. Added FREETEXT_VERBS and
exempted them.

The second one ran quieter. trim_md stripped `*` off both ends of every arg to
clean markdown bolding, so `complement perfect_cuboid_proof*` executed as
`complement perfect_cuboid_proof`: the agent reasoned about an excited state the
tool never saw, and the transcript reads as though it had. `*` and `_` are PAIRED
emphasis, so they are now trimmed from an end only when the other end carries the
same mark. A lone trailing `*` is a state marker and stays.

Third, `imasm prove` now accepts a catalog entry, which is what the agent kept
asking for. It only took a registered tool or an opcode word, so `imasm prove
perfect_cuboid_proof` fell through to a syntax error. But the strange loop makes
this exact: the 49 Shavian types are themselves full IMASM programs under
ob3ects/primitives, so a 12-tuple IS twelve programs composed in canonical order.
expand_entry reads the entry's own glyphs, names the types, unfolds each, and
carries the fork/fuse pairs across at their offsets. Nothing invented.

The glyph→type-name table is generated, not typed: matched from the canonical
enums to the names ob3ects/primitives actually uses, 49/49, no conflicts —
confirming the naming is glyph-global. Three spellings diverge for reasons worth
recording: the alphabet says `if`, `or`, `ha_ha`; Python must say `if_`/`or_`
because they are keywords and Lean escapes them `if'`/`or'`.

Worth saying plainly: the golem behaved correctly throughout. It refused to claim
the reagent existed — "the imscribe call was emitted but its direct output was
not returned; we have no ground-truth confirmation" — and held at B. MUST ONLY
speak what a tool grounds did its job while the tool layer was gagging it. Tests
pin all three.
