Close two golem leaks: no-op prod for tool-less drafts, and verdict must match the numbers

Ran the operator on a big set with a "prove/characterize this" framing and watched it write
a whole "Execution: Structural Tools" section -- --scan-mediators, --polymerize, --close, all
of it -- as PROSE, without emitting a single TOOL: line, then close with "PROVED. The manifold
is closed." Nothing ran. The loop had nothing to iterate on, and the golem constraint can only
bind tools that actually executed, so the entire section and the verdict were confabulated.
That is the residual I had flagged: a proof framing lets the operator skip the tools entirely.

Two fixes in ask_native/src/main.rs, both in the ACT->OBSERVE loop:

1. NO-OP PROD GUARD. Before the loop, if the draft ran ZERO tools (no TOOL: line) but it
   narrated structural work -- new helper mentions_structural_work() checks for verb stems
   (polymeriz, arrange, mediator, excite, enchain, cycliz, modulus, pathway) and --flags, or
   the question itself carries a `--` flag -- prod it ONCE: "you named polymerize/close/scan
   but ran zero tools; you may not assert a computed result you did not compute; emit TOOL:
   lines now (verb without dashes, catalog names as args), then answer grounded in what they
   return." The prod turn becomes the new round-0 action and the loop proceeds on it.
   Observed live: a prompt that said "prose only, do NOT emit any tool calls" produced a
   tool-less draft, the guard fired, and the operator then ran arrange/polymerize/close/
   material for real -- the harness insisting on grounding over an explicit no-tools order.

2. VERDICT MUST MATCH THE NUMBERS. Even when tools DID run and every one came back "terminated
   early / 1 unit 0 bonds / linear / telechelic," the operator still declared "PROVED (a
   Branched Heteropolymer)" -- redefining PROVED to dodge "never contradict a tool," and
   naming an architecture the counts (1 unit, 0 bonds -- not a polymer) flatly refute.
   Strengthened the loop's UPDATE instruction: the verdict must match the DP/bond/cyclization
   numbers -- if everything terminated or came back non-cyclic the assembly does NOT close, so
   do not call it closed/complete/a ring/PROVED, and do not name an architecture the counts do
   not support; report that it does not close, and which arrangement (if any) the tools showed
   would. Observed live on the same essay-mode set: the draft's "Verdict: PROVED" got corrected
   in OBSERVE/UPDATE to "NOT PROVED (Terminated). Does not close. Not a macrocycle, network, or
   polymer. Status: F."

Known residual, not chased here: the ManuscriptSpine banner still reads fused=T even when the
grounded answer is F/B -- the spine grades the answer's coherence, it does not yet ingest the
tools' closure result. And the architecture wording still slips occasionally ("branched"
against a linear result). Both are smaller than the two leaks closed here.
