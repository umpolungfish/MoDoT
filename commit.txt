Wire the type-router into the agent breath (proof turns reach the kernel)

The router was built and tested but it sat beside the organism, not in it. The
breath ran every turn through the ManuscriptSpine (vessel co-typing plus witness
scaffold) and never touched the prover, so a proof-shaped question typed at the
live agent never reached the Lean kernel. This mounts it.

I put a proof-intent gate at the top of _think. A turn is a formal proof request
iff it opens with `prove:` or carries a literal Lean theorem or lemma declaration.
Those turns divert to the router (imscribe, fold to N or T-F or B, dispatch); the
arm's output becomes the thought content and seeds the kernel value. Everything
else (proof_intent returns None) flows through the ordinary conversational breath
unchanged, so no chat turn is ever sent to Lean. The gate is conservative on
purpose.

The verdicts follow the ob3ect. Kernel closure is T and the content is the
machine-checked proof. An unclosed goal is held as B, a navigation frontier, never
a verdict of unprovability. A routed dialetheia is B with the Witness scaffold. A
vacuous goal is N.

Integration, not a second pipeline. The agent owns one TypeRouter over its own
LLM (self.router), the proof turn commits a proof record to the Crystal FS beside
the thought and spine records, and a public prove() method exposes the same path
programmatically. The router import is lazy inside __init__ because
agent to router to prover to agent would cycle at module load.

Verified live. Through the breath, `prove: theorem wired_demo : (2 : Nat) + 2 = 4`
routes T to the kernel loop and closes green on the first pass with source=router,
the proof returned as rfl. proof_intent leaves ordinary chat alone. The router,
vessel, and spine selftests still pass, and the scratch module is restored to its
green placeholder so lake build Imscribing stays clean.
