Fix the witness catalog in ask_native: canonical file, live merge, real scoring

`./ask` was matching questions against the stale mOMonadOS/IG_catalog.json copy
(5264, unsynced) instead of the canonical imscribing_grammar/IG_catalog.json
(5275), and never merged in the live-crawler catalog at ~/.imscrbgrmr/catalog.json
the way `imscribe catalog list` does — 5292 real entries down to 5264. Fixed the
catalog preference in both the `ask` wrapper and Rust's own find_catalog, and
ported the live-crawler merge into load_catalog.

Also ported the diacritic/anchor fix I'd only made on the Python side:
normalize() now folds ö/ő/ø/ü/é/è/á and the math-unicode symbols before matching,
and the anchor list picked up the foundational/self-reference terms (godel, liar,
tarski, halting, cantor, diagonal, epimenides, ...) so a self-referential question
actually reaches liar_paradox / godel_incompleteness_theorem / epimenides_paradox
instead of matching nothing.

Found and fixed two scoring bugs while verifying this end to end:
- A single generic glossary-style catalog entry ("argument", "theorem", ...) could
  outscore a real compound match just because its one word appears as a substring
  anywhere in a long question — Cantor's diagonal argument was matching the bare
  `argument` entry instead of `classical_cantor_diagonal`. The big containment
  bonus now requires a compound name or a short/near-exact query.
- Rust's tokenizer never dropped stopwords the way the Python side does, so
  common words like "the"/"all"/"are" inflated totally unrelated entries. Worse,
  short name parts ("pi", "eta", "the") were scored by raw substring containment,
  so "eta" matched inside "cretans" and "pi" matched inside "epimenides" — an
  Epimenides-paradox question came back with `pi` as the primary witness. Added
  the stopword list and required exact token matches for name parts <=3 chars.

Verified against the regression set (liar_paradox, tarskis_undefinability,
godel_incompleteness, halting_problem, collatz_conjecture, erdos_hajnal,
hadwiger_conjecture, cantors_paradox, continuum_hypothesis) plus the two cases
that surfaced the bugs (classical_cantor_diagonal, epimenides_paradox) — all
resolve to the right primary witness now. The `prove:` kernel path is untouched
and still closes rfl goals through lake build the same as before.
