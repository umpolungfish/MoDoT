Give the eagle a way to fix a fumbled verb call

run_structural_tool returned None for two unrelated cases: a verb that
does not exist, and a real verb given too few names. The agent loop
collapsed both into one line, "not an available verb / missing args", so
a real call like `polymerize A` (one name) or `scan A` (one name) was
reported as if polymerize and scan did not exist. With no correct form to
aim at, the agent re-emitted the same bad call round after round, burning
eagles on what was really just a missing second name.

Now the two cases are split. verb_usage(verb) holds the canonical call
form for every real verb (wording taken from the TOOLS_PROMPT the agent
already reads). tool_miss_message uses it: a real verb given bad args gets
"wrong or too few args (you gave: ...). Correct form: polymerize M1 M2...;
2+ names to chain", and only a genuinely unknown verb gets "not an
available verb" plus the list of the real ones. Either way the next round
has something to correct toward.

STRUCTURAL_VERBS is the single source of truth for the verb list, and a
test asserts every one of them has usage help, so a real verb can never
again be reported as nonexistent. Further tests cover the under-argged
path (returns None), the actionable message text, and the unknown-verb
list.
