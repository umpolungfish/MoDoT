MoDoT: an echo of the manual is not a call

The prompt documents `TOOL: imscribe <name> [description]`. The model quoted
that line back while explaining itself, and the extractor could not tell a
citation from a directive -- so it DISPATCHED the placeholder. A live run
spent a real mint trying to imscribe an entry named `NAME`, and had the
generator been healthy it would have written `NAME` into the catalog: the
manual's own placeholder, minted as an object.

The placeholders are the tell. A real argument is never `<name>`,
`[description]`, or a bare `NAME` shout. Refusing them costs nothing, since
no real call looks like one -- checked before the FREETEXT bypass, because
imscribe is precisely the verb whose template gets quoted. Tests cover both
directions: the echo does not dispatch, and a real free-text imscribe with a
prose description still runs.
