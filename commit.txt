MoDoT: tell the agent what it can actually type

`close` found the closing monomer, printed the exact next step, and the agent
spent eagle after eagle never running it. It was not blind. The next step was
printed as `./ask --polymerize clink_l9 sixteen_3_trilattice
bootstrap_stage_2_topology_corrected turbulent_flow` -- a SHELL command. The
agent's only way to act is a `TOOL:` line; there is no path from its interface
to that string. So the tool handed it the answer in a language it cannot
speak, and the run that followed read as the agent ignoring what it was shown.
A human ran that one line and the ring closed.

Five sites printed shell dialect into agent-facing output: three polymerize
next-steps and two SET/radical-ion routes.

Now the suggestion renders in the dialect of whoever is reading it --
`TOOL: polymerize a b c` for the agent, `./ask --polymerize a b c` at a shell.
Same act, two registers. run_structural_tool arms it: everything reached
through the dispatcher is being read by the agent. Tests cover both
directions, plus that the dispatcher arms the flag.

A next step the reader cannot execute is not a suggestion.
