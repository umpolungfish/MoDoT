Add material registry: register / recall / export

The persistence pair, the last of the material-algebra roadmap. A material is a
ring of ordered units plus its computed sheet, not a single tuple, so it gets its
own store rather than being collapsed into a catalog chimera.

  --forge … --register NAME   forge the ring, then upsert its full sheet (ordered
                              units, spectrum, rho, conductance, strain, energy,
                              weakest bond, branched) into materials.json, a
                              MoDoT-local named library.
  --recall NAME               reload a registered material by name and print its
                              stored sheet, no respecifying from units. Lists the
                              registry when the name is missing.
  --forge … --export PATH     write the same record to a standalone portable file.
  --recall NAME --export PATH  re-export an already-registered material.

Also the `register NAME M1 M2…` and `recall NAME` agent verbs. The registry path
resolves by $MOMONADOS_MATERIALS, else the MoDoT root (binary-relative), else the
working dir; materials.json is runtime state, not tracked.

Verified end to end: register a branched 3-ring, recall it (rho=2.7321,
CONDUCTIVE, spectrum recovered intact), export a 4-ring to a JSON file.
