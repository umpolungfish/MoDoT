Wire MoDoT as a single ManuscriptSpine pipeline (not parallel arms)

The breath runs one end-to-end spine, the runtime analogue of Lean
VAE_Vita_ManuscriptSpine.manuscript_formal_spine:

  spine.prepare(question)   # IMSCRIB: demand imscription + catalog scaffold
  LLM answer                # FSPLIT
  spine.complete(q, a, v)   # EVALT/EVALF + FFUSE model x vessel + IFIX

modot/spine.py owns DualLinkVessel + witness_proof; agent.spine is that object,
and selectivity is an alias to the same instance. The Crystal record type `spine`
packages the prove/unify/port/witness/transport faces; `vessel` is now a thin log
alias. Self-test: python3 -m modot.spine

This supersedes the classical selectivity grader (MUST/MUSTNOT schemas, a 0.6
threshold, LLM-graded correctness, a protocol == integration check), which is
removed. Verification is imscription: demand and answer are imscribed and co-typed
in the d=12 Dual-Link SIC frame, and conflict is held as B.
