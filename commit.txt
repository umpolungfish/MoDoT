Race a portal-guided attempt against standard decomposition in the prover

This is the chrysopoeia_2048 portal move (ig-docs/manuscripts3), generalized
past SIC moduli: don't always walk the full decomposition distance to a proof.
If an already-catalogued witness sits structurally close to the goal, cite it as
a hint and race that attempt against the standard one. The kernel is the sole
discriminator — no separate exact-match sieve needed, because a bad hint just
fails to compile, and `lake build` costs little enough at this budget to just
try both and see which (if either) closes.

portal_hint() (ask_native/src/prover.rs) reuses the same search_catalog scoring
I fixed in the witness-catalog commit — this is a hint only, not a claim the
witness is independently proven (checked directly: the real catalog carries no
populated proved_hint/structural_algebra field to pre-filter on, so lexical/
structural match score is genuinely all there is to go on here).

race_portal() spawns two threads at a modest, bounded budget (flat=3, fuse=2,
depth<=1 — cheap enough that losing the race costs little): one standard, one
with the goal augmented by the portal hint. Each targets its own scratch module
(Imscribing.Scratch.ProverScratch / ProverScratchB, both now registered in
p4ramill's lakefile.toml) so the two `lake build`s don't collide on the same
file — confirmed safe by running both concurrently by hand first (two full
lake builds against distinct modules in the same project, both green, no lock
contention). Whichever side closes wins; if neither does within the tight
budget, prove() falls through to the existing 3-round escalation loop
unchanged, so this is strictly additive — no regression path.

Pulled the definitional-rigging guard (Guard 2, already in the standard path)
out into apply_rigging_guard() so both the race and the escalation loop check
it identically — a portal-guided proof that "wins" by defining the goal's own
vocabulary is exactly as disqualified as a standard one that does.

Verified: a goal with no strong catalog match behaves identically to before
(no portal line printed, straight to escalation). A goal lexically close to
`collatz_conjecture` triggers the race, both threads run genuinely concurrently
(interleaved PROVE(d0) lines), both scratch files are restored to placeholder
after, and the winning side is reported by name in the proof's note.
