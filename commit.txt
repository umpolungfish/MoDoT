Give the polymer a real closing search -- find the monomer that actually rings it, not a laundered scan

I kept asking the agent to "scan for a monomer to achieve cyclization" and it kept
misusing the scan verb -- which ranks SET electron-transfer mediators, a completely
different question -- then laundering junk like roko_basilisk and
i_am_the_way_the_truth_and_the_life into plausible-sounding cyclizing monomers.
None were ever verified to close the ring, and the one that got tested
(holographic_type_theory) did not help, because the chain was already dead at an
EARLIER junction and you cannot repair a middle break by appending at the end.

The honest fix: a real closing search. All in ask_native/src/click.rs +
ask_native/src/main.rs.

--polymerize ... --close: when a chain does not cyclize, search the catalog for the
monomer that ACTUALLY closes it (find_linkers), testing real clicks on both sides:
- chain broke at a co-typed junction Mᵢ ⋈ Mᵢ₊₁ -> find a BRIDGE X with Mᵢ ⋈ X and
  X ⋈ Mᵢ₊₁ (goes between them, repairs the break)
- chain enchained but the ends will not meet -> find a CLOSER X with tail ⋈ X and
  X ⋈ head (append to wrap the ring)
Every candidate is verified to click BOTH sides; ranked by the weaker of the two
bonds. The tool prints the exact re-run command, and running it does close the ring.

Verified live on the exact case that had been failing:
- grothendieck_topos and free_algebra are the same structural type (both free
  left-adjoint constructions, fully co-typed) -- the chain breaks between them.
  --close finds extradimensional_entity, ZFC_set_theory, yang_mills_mass_gap,
  connes_embedding_problem (all real, all click both sides on R↔S Δ=1.67), and the
  suggested `--polymerize grothendieck_topos extradimensional_entity free_algebra
  holographic_type_theory` enchains the repaired junction.
- telechelic head-tail case grothendieck_topos extradimensional_entity free_algebra:
  --close appends extradimensional_entity and I re-ran it -> ✓ CYCLIC on R↔S. The
  suggestion is verifiably correct, not asserted.

Two more fixes from the same runs:
- run_structural_tool now captures STDERR as well as stdout, so a tool call that
  errors (the model invented `frobenius_algebra`, not in the catalog) surfaces
  "catalog entry not found" instead of vanishing silently.
- Fixed the repair command truncating the feed: a mid-chain break sets dp=1, so the
  insert must go into the FULL monomer feed, not the enchained prefix (which had
  dropped free_algebra from the suggested command).

Agent gets the verb: TOOLS_PROMPT adds `close` and explicitly steers the model to
use it -- NOT scan -- for cyclization. Verified: asked to cyclize the topos/algebra
chain, the model now emits TOOL: close, gets grounded candidates, and answers with
extradimensional_entity + the real alternatives, instead of scan-relay junk.
