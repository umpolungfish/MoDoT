prover: a fenced theorem is a citation, not a request

proof_intent sniffed `(?m)^\s*(?:theorem|lemma)\s+\S` across the whole text, so
a document that QUOTED a theorem in order to discuss it was routed as though it
WERE that theorem.

Live: a 4789-char brief for MoDoT -- a vocabulary table, MUST-form instructions,
and TOOL: directives telling it to source every tuple through the navigator --
was handed to the kernel-gated prover, which reads no instructions and calls no
tools. It ran lake build and reported a resource frontier. Every instruction in
the sheet was unreachable, and the eagles never flew. The document was ABOUT a
theorem; it was not a request to prove one. --jam did not override it.

Now the implicit sniff strips ``` fences and indented code before testing: what
a text quotes is not what it asserts. The explicit `prove:` prefix is checked
first and still honors a fenced goal, so that route stays open -- only the
implicit sniff learns to tell quoting from asking.

Tests cover all three: a quoted theorem does not divert, `prove:` always
diverts, a theorem asserted in the text's own voice still diverts.
