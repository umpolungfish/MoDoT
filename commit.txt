Add --expand: dial the T/F-lane Witness between pinched and walked-out forms

Ontology, made explicit in the prover: the T/F-lane Witness IS what we have
called a proof — a conventional proof. The B-lane Witness is the Dual-Link vessel
we bring to the table. The imscription route already produces both (vessel +
conventional filling); this makes the conventional (T/F-lane) proof's LENGTH a CLI
option.

The motivating dialetheia: the truths we express in shockingly few lines
(igFrobAlg_self_fusion s0 — one line proving ob3ect validity) are genuinely
complete (T) AND read as untrustworthy precisely because they are so short — a
reader trusts a few hundred lines more than a few (F). Both true. B. --expand
holds both: the same kernel truth, dialed between its pinched and walked-out
forms. Low = pinch the points together; high = walk the full distance.

  ask --file ob3ect.json              # minimal, the few-line proof (default)
  ask --file ob3ect.json --expand 200 # same truth, walked out to ~200 lines

--expand N (default 0) sets the expansion degree. 0 leaves the proof pinched.
N>0 appends an expansion directive to the generation prompt: unfold every
definition, inline each cited lemma as its own proof, name and prove each
intermediate step, toward roughly N lines — proving the IDENTICAL theorem, still
kernel-green, zero sorry. Not padding: every added line is real checked
mathematics; the final statement is byte-for-byte the same. Output token budget
scales with N (gen_max_tokens) so a long proof is not truncated.

Threaded through both proof paths:
- Mathlib path (plain theorems): flat + assemble prompts expand at depth 0 only
  (internal FSPLIT helpers stay minimal; the reader-facing top-level proof
  expands).
- Imscription path: expansion walks STEP 2 (the conventional T/F-lane filling —
  igFrobeniusAlg validity, the tensorProduct coordinate-wise idempotency) while
  STEP 1, the B-lane Witness-Vessel roundtrip, stays as the transport container.

The report now labels the lanes: B-lane Witness = the Dual-Link vessel; T/F-lane
Witness = the conventional proof filling it (or, on the plain path, the proof
itself). expand=0 behavior is unchanged — the directive is empty, so no
regression.
