Cache tool results so a repeated batch never re-executes

The loop re-ran identical calls every round until the stall guard broke it
after 3 rounds — the transcript shows rounds 3/4/5 executing the same
close/sublime/filter/ascend batch three times. A structural verb is a pure
function of its args, so re-running only burns work and duplicates output.

ran_signatures is now ran_results (signature -> output): a call whose exact
signature already ran returns the CACHED result and is not re-executed;
only genuinely new calls hit run_structural_tool. Stall detection now keys
off the cache, so an all-cached round still closes the loop.

Also: prose guard on extract_tool_calls. The model narrates next-steps as
TOOL: lines ("TOOL: primitive_peel on the 6 survivors", "TOOL: ascend
again", "TOOL: dope operation (format …)"), which were run on garbage args
("the", "again", "operation"). A call is skipped when its first arg is an
English stopword or any arg carries prose punctuation ( ( ) ` : ), so only
real catalog-token calls run. Tests pin both. Suite green (33 passed).
