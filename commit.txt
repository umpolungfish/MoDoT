I made the composer surface carry its own manifest.

Opening imasm_composer.html directly (file://) failed to fetch
ob3ects/imasm_manifest.json because browsers block file:// fetches, leaving
only the picker or a local server. Now `imasm export` splices the manifest
JSON straight into an application/json block inside the page (with </ escaped
so the data can never close its own script tag, and idempotent across
re-exports), and the page reads the inline block first, falling back to the
fetch and the picker only when it has not been exported into yet. The surface
was always fully local and self-contained; now its data is too.

Verified: export embeds 340 tools and 12 opcodes, the inline block parses
back, and a second export re-splices cleanly.
